# HR PDF Sorter

Утилита для автоматической раскладки PDF-файлов платежных документов по папкам сотрудников.

Скрипт анализирует имя PDF-файла, извлекает ФИО и подбирает наиболее подходящую папку по совпадению слов и fuzzy-score.

## Что делает

- ищет PDF-файлы в указанной рабочей папке;
- ищет папки сотрудников в этой же рабочей папке;
- извлекает ФИО из имени файла вида `ПП от 02.10.2025 Иванов Иван_rus.pdf`;
- подбирает папку сотрудника;
- в режиме проверки показывает план перемещения;
- в боевом режиме перемещает PDF в найденную папку;
- не перемещает файл, если совпадение слабое или неоднозначное;
- при конфликте имен добавляет `(2)`, `(3)` и так далее.

## Структура рабочей папки

```text
для HR/
├─ Аббасов Азер/
├─ Абдразак Мухаммед-Шариф/
├─ Абдукадиров Бахтиёр/
├─ Легализация Декабрь 2025.xlsx
├─ ПП от 02.10.2025 Абдразак Мухаммед-Шариф_rus.pdf
├─ ПП от 11.11.2025 Аббасов Азер_rus.pdf
└─ sort_payment_pdfs.py
```

## Проверка без перемещения

```bat
python sort_payment_pdfs.py --dir "C:\Users\Maksim\Desktop\для HR"
```

Или если скрипт лежит прямо в рабочей папке:

```bat
python sort_payment_pdfs.py
```

## Реальное перемещение

```bat
python sort_payment_pdfs.py --dir "C:\Users\Maksim\Desktop\для HR" --move
```

## Настройка чувствительности поиска

По умолчанию минимальный порог совпадения — `0.78`.

```bat
python sort_payment_pdfs.py --min-score 0.82
```

Чем выше значение, тем осторожнее скрипт. Для HR-документов лучше держать порог в диапазоне `0.78–0.88`.

## Создание CSV-отчета

```bat
python sort_payment_pdfs.py --report report.csv
```

В отчете будут статусы:

- `DRY_RUN` — найдено совпадение, но файл не перемещался;
- `MOVED` — файл перемещен;
- `SKIPPED_NOT_FOUND` — подходящая папка не найдена;
- `SKIPPED_AMBIGUOUS` — совпадение неоднозначное;
- `ERROR` — ошибка при перемещении.
