# Описание скриптов

Документ фиксирует назначение, входные данные, выходные данные и безопасный порядок запуска для основных утилит репозитория.

## `sort_payment_pdfs.py`

### Назначение

Раскладывает PDF-файлы платёжных документов по папкам сотрудников. Основной источник ФИО — имя PDF-файла.

Пример имени:

```text
ПП от 02.10.2025 Иванов Иван_rus.pdf
```

### Как работает

1. Ищет PDF-файлы в рабочей папке.
2. Ищет папки сотрудников.
3. Чистит имя файла от служебных слов: `ПП`, `от`, `rus`, `pdf`, даты и похожего мусора.
4. Нормализует ФИО и названия папок.
5. Считает совпадение через токены и fuzzy-score.
6. В безопасном режиме показывает план перемещения.
7. В боевом режиме перемещает файл в найденную папку.

### Входные данные

- папка с PDF;
- папки сотрудников;
- имя PDF должно содержать ФИО или достаточно похожее написание ФИО.

### Выходные данные

- перемещённые PDF, если указан `--move`;
- консольный лог;
- CSV-отчёт, если указан `--report`.

### Основные команды

Проверка без перемещения:

```bat
py sort_payment_pdfs.py --dir "C:\Users\Maksim\Desktop\для HR"
```

Реальное перемещение:

```bat
py sort_payment_pdfs.py --dir "C:\Users\Maksim\Desktop\для HR" --move
```

Отчёт:

```bat
py sort_payment_pdfs.py --report reports\sort_report.csv
```

### Важные параметры

| Параметр | Назначение |
| --- | --- |
| `--dir` | Рабочая папка. По умолчанию текущая. |
| `--move` | Включает реальное перемещение. Без него только проверка. |
| `--min-score` | Минимальный порог совпадения. По умолчанию `0.78`. |
| `--ambiguity-gap` | Если лучший и второй кандидат слишком близки, файл пропускается. |
| `--recursive-pdfs` | Искать PDF рекурсивно. |
| `--recursive-folders` | Искать папки сотрудников рекурсивно. |
| `--report` | Путь к CSV-отчёту. |

### Когда использовать

Использовать, когда нужно разложить PDF по папкам, а сопоставление можно сделать по ФИО в имени файла.

### Когда не использовать

Не использовать для сверки PDF с Excel. Для этого есть `ru_decont.py`.

## `ru_decont.py`

### Назначение

Сверяет русские PDF-платёжки с Excel-реестром и дописывает извлечённые из PDF данные в найденные строки Excel.

Скрипт рассчитан на платёжные документы Ziraat Bankasi в старых и новых шаблонах, включая сканы и многостраничные PDF.

### Как работает

1. Открывает Excel.
2. Находит строку заголовков с обязательными колонками.
3. Строит индексы по `ФИО рус`, `Дата оплаты`, `Y.T.C № Кимлики`.
4. Собирает PDF из папки.
5. Для каждого PDF извлекает:
   - дату;
   - ИНН / кимлик;
   - фамилию / наименование;
   - имя;
   - суммы платежей.
6. Для текстовых PDF сначала использует `pdfplumber`.
7. Для сканов и проблемных PDF использует OCR через Tesseract.
8. Для многостраничных PDF определяет полезные страницы с таблицами и пропускает почти пустые страницы.
9. Сопоставляет PDF со строкой Excel.
10. Записывает результат в колонки начиная с `AH`.
11. Создаёт итоговый Excel и CSV-отчёты.

### Входные данные

Структура по умолчанию для BAT-запуска:

```text
hr-pdf-sorter/
├─ excel_input/
│  └─ input.xlsx
└─ pp_ru/
   ├─ *.pdf
   └─ ...
```

Обязательные колонки в Excel:

```text
ФИО рус
Дата оплаты
Y.T.C № Кимлики
```

### Выходные данные

Рядом с исходным Excel создаются:

```text
input_RU_decont_filled.xlsx
RU_decont_parsed_report.csv
RU_decont_unmatched_report.csv
RU_decont_ocr_debug_report.csv
```

### Запуск

Через BAT:

```bat
compare_ru_decont.bat
```

Вручную:

```bat
py ru_decont.py --excel "excel_input\input.xlsx" --pdf-dir "pp_ru"
```

С конкретным листом:

```bat
py ru_decont.py --excel "excel_input\input.xlsx" --pdf-dir "pp_ru" --sheet "Лист1"
```

Без OCR:

```bat
py ru_decont.py --excel "excel_input\input.xlsx" --pdf-dir "pp_ru" --no-ocr
```

### Колонки записи в Excel

| Колонка | Поле |
| --- | --- |
| `AH` | Дата |
| `AI` | ИНН / Кимлик |
| `AJ` | Фамилия / Наименование |
| `AK` | Имя |
| `AL` | Сумма |
| `AM` | Сумма 2 |

### Логика сопоставления

Приоритеты:

1. `ФИО + дата + кимлик`;
2. `ФИО + дата`, если найдено одно совпадение;
3. `ФИО + кимлик`, если найдено одно совпадение;
4. только `ФИО`, если сотрудник в Excel встречается один раз.

Если совпадение неоднозначное или отсутствует, Excel не заполняется. Запись уходит в `RU_decont_unmatched_report.csv`.

### OCR-особенности

Скрипт умеет:

- читать сканированные PDF;
- находить дату в верхнем правом блоке;
- находить ИНН / кимлик в блоке налогоплательщика;
- находить ФИО в вариантах `Фамилия / Наименование`, `Фамилия / Должность`, `Фамилия / Звание`;
- извлекать суммы из таблицы платежа;
- пропускать почти пустые страницы в многостраничных PDF;
- фиксировать полезные и пропущенные страницы в отчётах.

### Диагностика

Если PDF не сопоставился, открыть `RU_decont_unmatched_report.csv` и проверить:

- `Дата`;
- `ИНН / Кимлик`;
- `Фамилия / Наименование`;
- `Имя`;
- `match_key_name`;
- `match_mode`.

Если OCR подозрительный, открыть `RU_decont_ocr_debug_report.csv` и проверить:

- `raw_top_date_ocr`;
- `raw_kimlik_ocr`;
- `raw_name_ocr`;
- `payment_pages`;
- `skipped_pages`.

### Частые причины `unmatched`

1. ФИО в Excel написано иначе, чем в PDF.
2. В Excel другая дата оплаты.
3. В Excel другой кимлик.
4. В PDF новый шаблон, под который ещё не настроены OCR-кропы.
5. Имя файла повреждено кодировкой, а OCR ФИО не смог прочитать.

## `compare_ru_decont.bat`

### Назначение

Упрощённый запуск `ru_decont.py` на Windows.

Содержимое команды:

```bat
py ru_decont.py --excel "excel_input\input.xlsx" --pdf-dir "pp_ru"
```

### Когда использовать

Когда рабочая структура стандартная:

```text
excel_input\input.xlsx
pp_ru\*.pdf
```

### Когда править BAT

Править, если:

- Excel лежит в другом месте;
- папка PDF называется иначе;
- нужно указать конкретный лист Excel;
- нужно временно отключить OCR.

Пример с листом:

```bat
py ru_decont.py --excel "excel_input\input.xlsx" --pdf-dir "pp_ru" --sheet "Лист1"
```

## Рекомендуемый рабочий процесс

1. Сложить PDF в рабочую папку.
2. Сначала запустить безопасный сценарий или сверку без удаления исходников.
3. Проверить CSV-отчёты.
4. Исправить Excel или OCR-кропы, если есть `unmatched`.
5. Только после проверки использовать результаты в рабочем процессе.

Код должен ехать в Git, персональные документы — нет.
